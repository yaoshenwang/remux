# ADR: Runtime Identity Terminology

Date: 2026-03-30
Status: Accepted

## Context

Remux is evolving from a terminal cockpit (monitor and control tmux/zellij sessions) into a workspace OS that will eventually support collaboration, agent execution, and structured artifacts. The current codebase mixes several naming conventions inherited from early prototypes — "Scroll mode", "WorkspaceSnapshot", and overloaded uses of "Workspace" — creating confusion about what each concept actually means at the runtime layer versus the future collaboration layer.

Fixing terminology now, before the API surface and user-facing strings multiply further, avoids a much larger rename tax later and ensures that new features land with consistent naming from day one.

## Decision

### Primary product terms

| Term | Scope | Definition |
|------|-------|------------|
| **Runtime** | Core | The running instance that owns PTYs, sessions, tabs, and panes. Not "Workspace" at this layer. |
| **Inspect** | Core | The product feature for reviewing terminal history (scrollback content, search, etc.). Replaces "Scroll" / "Scroll mode" as a product surface name. |
| **Live** | Core | Real-time terminal interaction — keystrokes flow to PTY, output streams to client. |
| **Control** | Core | Session, tab, and pane management operations (create, rename, close, reorder). |
| **RuntimeSnapshot** | Core | Serialised state of a runtime (metadata + pane buffers). Replaces "WorkspaceSnapshot" as the primary name. |
| **Topic** | Future | A collaboration context unit — a thread of work that groups terminals, artifacts, and discussion. |
| **Run** | Future | A single agent execution unit within a topic. |
| **Artifact** | Future | A structured output or product generated by a run (file, diff, report, etc.). |
| **Approval** | Future | A gate that pauses agent execution until a human approves a side-effect. |
| **Agent** | Future | An AI execution member that can perform runs inside a topic. |

### Banned terms

These terms must **not** be used as product-facing names in new code, UI strings, or documentation:

| Banned | Replacement | Reason |
|--------|-------------|--------|
| "Scroll" / "Scroll mode" (as a product surface name) | **Inspect** | "Scroll" describes a low-level mechanic, not a user intent. "Inspect" communicates the purpose. |
| "WorkspaceSnapshot" (as primary name) | **RuntimeSnapshot** | The snapshot captures runtime state, not workspace-layer (team/org) state. `WorkspaceSnapshot` is kept **only** as a deprecated wire-compatibility alias in serialisation code. |
| "Workspace" for runtime-level concepts | **Runtime** | "Workspace" is reserved for the future collaboration layer — team/org containers that hold multiple runtimes, topics, and membership. Using it at the runtime level causes semantic collision. |

### Technical terms still allowed

The following uses of otherwise-banned vocabulary remain acceptable because they refer to established terminal-standard concepts, not Remux product surfaces:

| Term | Where allowed | Rationale |
|------|---------------|-----------|
| `scrollback` | xterm.js configuration options and terminal buffer internals (e.g., `scrollback: 10000`) | This is a de-facto standard term in terminal emulators (xterm, VTE, Windows Terminal). Renaming it would contradict the ecosystem. |
| `--scrollback` | CLI flag (e.g., `remux --scrollback 5000`) | Inherited from the current public CLI interface. Will be renamed in a separate CLI migration (tracked independently) to avoid a breaking change in this cycle. |

## Consequences

Positive:

- every new feature, API field, and UI label lands with a name that scales to the workspace-OS vision
- "Runtime" vs "Workspace" split eliminates the most common source of naming ambiguity in code reviews
- the Inspect / Live / Control vocabulary maps directly to user intents, improving discoverability

Tradeoffs:

- existing code and tests require a rename pass (tracked in the runtime-identity-cleanup branch)
- the deprecated `WorkspaceSnapshot` wire alias must be maintained until all persisted snapshots have been migrated
- contributors familiar with the old names will need a brief adjustment period
